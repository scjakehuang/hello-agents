#### 考察一：RAG技术理解与幻觉成因

**问题1：** 解释RAG的工作原理，并说明它如何帮助减少AI幻觉。

**RAG减少幻觉的原理：**

| 原理 | 说明 |
|------|------|
| 事实约束 | LLM必须基于检索到的真实文档生成答案 |
| 可溯源 | 每个答案都有对应的来源文档作为支撑 |
| 实时更新 | 知识库可随时更新，无需重新训练模型 |
| 减少编造 | 有具体材料参考，LLM不需要凭空生成 |

**问题2：** 在RAG系统中，幻觉可能来源于哪些环节？

async def hybrid_search(query, top_k=10):
    # 1. 向量检索
    vector_results = await milvus.search(
        embedding(query),
        top_k=top_k * 2
    )
    
    # 2. 关键词检索
    keyword_results = await elasticsearch.search(
        query=query,
        top_k=top_k * 2
    )
    
    # 3. RRF融合
    def reciprocal_rank_fusion(results, k=60):
        scores = {}
        for rank, doc in enumerate(results):
            doc_id = doc['id']
            scores[doc_id] = scores.get(doc_id, 0) + 1 / (k + rank + 1)
        return sorted(scores.items(), key=lambda x: x[1], reverse=True)
    
    return rrf_fusion(vector_results, keyword_results)[:top_k]
```

def filter_results(results, threshold=0.7):
    """过滤低相关性结果"""
    filtered = []
    for doc in results:
        if doc['score'] >= threshold:
            # 去重
            if not any(d['id'] == doc['id'] for d in filtered):
                filtered.append(doc)
    return filtered
```
