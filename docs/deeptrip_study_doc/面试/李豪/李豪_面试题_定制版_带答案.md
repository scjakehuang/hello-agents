### 四、模型服务调用优化（5分钟）

| 问题 | 参考答案 |
|------|----------|
| 自定义TCP协议的优势？ | **TCP vs HTTP对比：**<br><br>| 对比项 | 自定义TCP | HTTP |\n|--------|-----------|------|\n| 协议开销 | 极低 | 较高（Headers） |\n| 连接复用 | 原生支持 | 需Keep-Alive |\n| 性能 | 高 | 中等 |\n| 调试难度 | 高 | 低 |\n\n**协议设计：**<br>```\n| 4字节长度 | 1字节类型 | N字节数据 |\n└──────────┴──────────┴──────────┘\n```\n\n- 长度：标识数据包总长度<br>- 类型：请求/响应/心跳<br>- 数据：序列化后的业务数据 |
| Python多进程突破GIL？ | **GIL限制：**<br>- Python GIL导致同一时刻只有一个线程执行Python字节码<br>- 多线程无法利用多核CPU<br><br>**多进程方案：**<br>```python\nfrom multiprocessing import Process, Queue\n\ndef worker(queue):\n    while True:\n        task = queue.get()\n        result = process(task)\n        queue.put(result)\n\n# 启动N个进程\nprocesses = [Process(target=worker, args=(queue,)) for _ in range(cpu_count)]\nfor p in processes:\n    p.start()\n```\n\n**进程间通信：**<br>- Queue：进程安全队列<br>- Pipe：双向管道<br>- 共享内存：高效大数据传输 |
| QPS 200到2000+的关键优化？ | **关键优化点：**<br><br>1. **协议优化**<br>   - HTTP → 自定义TCP二进制协议<br>   - 开销减少50%+<br><br>2. **连接池管理**<br>   - 预创建连接，避免频繁建连<br>   - 连接复用率提升<br><br>3. **DNS负载均衡**<br>   - 本地缓存DNS结果<br>   - 减少DNS解析延迟<br><br>4. **异步调用**<br>   - 非阻塞IO<br>   - 提升并发能力<br><br>5. **多进程突破GIL**<br>   - 充分利用多核CPU |

import java.util.*;
import java.util.concurrent.*;
import java.util.concurrent.atomic.AtomicInteger;

public class FeatureComputeEngine {
    
    private final ExecutorService executor;
    private final int timeoutMs;
    
    public FeatureComputeEngine(int threadPoolSize, int timeoutMs) {
        this.executor = new ThreadPoolExecutor(
            threadPoolSize,
            threadPoolSize,
            60L, TimeUnit.SECONDS,
            new LinkedBlockingQueue<>(1000),
            new ThreadFactory() {
                private final AtomicInteger counter = new AtomicInteger(0);
                @Override
                public Thread newThread(Runnable r) {
                    Thread t = new Thread(r);
                    t.setName("feature-compute-" + counter.incrementAndGet());
                    t.setDaemon(true);
                    return t;
                }
            },
            new ThreadPoolExecutor.CallerRunsPolicy()
        );
        this.timeoutMs = timeoutMs;
    }
    
    public Map<String, Object> computeFeatures(List<FeatureTask> tasks, long timeoutMs) {
        if (tasks == null || tasks.isEmpty()) {
            return Collections.emptyMap();
        }
        
        // 1. 创建所有任务的Future
        Map<String, CompletableFuture<Object>> futureMap = new ConcurrentHashMap<>();
        
        for (FeatureTask task : tasks) {
            CompletableFuture<Object> future = CompletableFuture.supplyAsync(() -> {
                try {
                    return task.compute();
                } catch (Exception e) {
                    return new FeatureError(task.getName(), e);
                }
            }, executor).completeOnTimeout(
                new FeatureTimeout(task.getName()),
                timeoutMs,
                TimeUnit.MILLISECONDS
            );
            futureMap.put(task.getName(), future);
        }
        
        // 2. 等待所有任务完成
        CompletableFuture<Void> allFutures = CompletableFuture.allOf(
            futureMap.values().toArray(new CompletableFuture[0])
        );
        
        try {
            allFutures.get(timeoutMs + 100, TimeUnit.MILLISECONDS);
        } catch (TimeoutException e) {
            // 超时，返回已完成的结果
        } catch (Exception e) {
            // 其他异常
        }
        
        // 3. 收集结果
        Map<String, Object> results = new HashMap<>();
        for (Map.Entry<String, CompletableFuture<Object>> entry : futureMap.entrySet()) {
            try {
                Object result = entry.getValue().getNow(null);
                if (result != null) {
                    if (result instanceof FeatureError) {
                        results.put(entry.getKey(), "ERROR");
                    } else if (result instanceof FeatureTimeout) {
                        results.put(entry.getKey(), "TIMEOUT");
                    } else {
                        results.put(entry.getKey(), result);
                    }
                }
            } catch (Exception e) {
                results.put(entry.getKey(), "ERROR");
            }
        }
        
        return results;
    }
    
    public void shutdown() {
        executor.shutdown();
    }
    
    // 错误结果包装类
    private static class FeatureError {
        final String featureName;
        final Throwable error;
        
        FeatureError(String name, Throwable e) {
            this.featureName = name;
            this.error = e;
        }
    }
    
    // 超时结果包装类
    private static class FeatureTimeout {
        final String featureName;
        
        FeatureTimeout(String name) {
            this.featureName = name;
        }
    }
}

import java.util.*;
import java.util.concurrent.*;

public class RagSearchService {
    
    private final EmbeddingService embeddingService;
    private final VectorDbClient vectorDb;
    private final CacheClient cacheClient;
    private final ExecutorService executor;
    
    public RagSearchService(EmbeddingService embeddingService, 
                           VectorDbClient vectorDb,
                           CacheClient cacheClient) {
        this.embeddingService = embeddingService;
        this.vectorDb = vectorDb;
        this.cacheClient = cacheClient;
        this.executor = Executors.newFixedThreadPool(10);
    }
    
    public List<Document> search(String query, int topK, double threshold) {
        if (query == null || query.isEmpty()) {
            return Collections.emptyList();
        }
        
        try {
            // 1. 检查缓存
            String cacheKey = generateCacheKey(query, topK, threshold);
            List<Document> cached = cacheClient.get(cacheKey);
            if (cached != null) {
                return cached;
            }
            
            // 2. 向量化查询
            float[] queryVector = embeddingService.embed(query);
            if (queryVector == null) {
                return Collections.emptyList();
            }
            
            // 3. 向量检索（多取一些用于过滤）
            List<Document> rawResults = vectorDb.search(queryVector, topK * 2);
            
            // 4. 相似度阈值过滤
            List<Document> filteredResults = new ArrayList<>();
            for (Document doc : rawResults) {
                if (doc.getScore() >= threshold) {
                    filteredResults.add(doc);
                }
            }
            
            // 5. 截取topK
            List<Document> results = filteredResults.subList(
                0, 
                Math.min(topK, filteredResults.size())
            );
            
            // 6. 缓存结果（异步）
            final List<Document> finalResults = new ArrayList<>(results);
            CompletableFuture.runAsync(() -> {
                cacheClient.set(cacheKey, finalResults, 300); // 5分钟缓存
            }, executor);
            
            return results;
            
        } catch (Exception e) {
            // 异常处理
            return Collections.emptyList();
        }
    }
    
    private String generateCacheKey(String query, int topK, double threshold) {
        return String.format("rag:%s:%d:%.2f", 
            DigestUtils.md5Hex(query), topK, threshold);
    }
    
    public void shutdown() {
        executor.shutdown();
    }
}

| 3 | 架构层面如何避免？ | **架构改进：**<br><br>**1. 熔断机制**<br>```java\nCircuitBreaker breaker = CircuitBreaker.builder()\n    .failureRateThreshold(50)\n    .waitDurationInOpenState(Duration.ofSeconds(30))\n    .build();\n```\n\n**2. 隔离机制**<br>- 线程池隔离：不同特征使用不同线程池<br>- 资源隔离：核心特征独立部署<br><br>**3. 限流机制**<br>```java\nRateLimiter limiter = RateLimiter.create(1000);\n```\n\n**4. 监控告警**<br>- 实时监控关键指标<br>- 设置合理的告警阈值 |
| 4 | 系统设计改进？ | **改进建议：**<br><br>**1. 异步化改造**<br>- 非核心特征异步计算<br>- 结果通过回调或消息返回<br><br>**2. 多级缓存**<br>```\n请求 → 本地缓存 → 分布式缓存 → 数据库\n```\n\n**3. 弹性伸缩**<br>- 基于QPS自动扩缩容<br>- K8s HPA配置<br><br>**4. 混沌工程**<br>- 定期故障演练<br>- 验证系统容错能力 |
