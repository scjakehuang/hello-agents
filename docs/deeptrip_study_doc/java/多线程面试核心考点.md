## 一、线程基础

### 1.1 线程状态

```
┌──────────────────────────────────────────────────────────────┐
│                    线程生命周期（6种状态）                      │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│                    ┌──────────┐                               │
│         ────────▶ │   NEW    │  新建，未调用 start()          │
│         start()   └────┬─────┘                               │
│                       │                                       │
│                       ▼                                       │
│                 ┌──────────┐                                  │
│          ┌────▶│ RUNNABLE │◀────┐                             │
│          │     │ (可运行)  │     │                             │
│          │     └────┬─────┘     │                             │
│          │          │            │                             │
│          │   ┌──────┼──────┐    │                             │
│          │   ▼      ▼      ▼    │                             │
│          │ ┌─────┐┌─────┐┌─────┐│                             │
│  notify/ │ │WAIT ││TIME ││BLOCK││   wait/                    │
│  notifyAll│ │ING  ││D_WA ││ED   ││   sleep/                   │
│          │ └──┬──┘└──┬──┘└──┬──┘│   join/                     │
│          │    │       │       │  │   park/                     │
│          │    │       │       │  │   获取锁失败                  │
│          └────┘       │       │  │                             │
│             wait()    │超时    │获取锁                          │
│             join()    │自动   │成功                             │
│             park()    │唤醒   │                                 │
│                       │       │                                │
│                       ▼       ▼                                │
│                    ┌──────────┐                                │
│                    │TERMINATED│  终止                           │
│                    └──────────┘                                │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

### 1.2 Thread 核心方法

| 方法 | 说明 | 注意 |
|------|------|------|
| start() | 启动线程 | 不要直接调 run()，那样只是同步调用 |
| sleep(ms) | 休眠，不释放锁 | Thread.sleep() 静态方法 |
| wait()/notify() | 等待/唤醒，释放锁 | 必须在 synchronized 块内 |
| join() | 等待线程结束 | 底层调 wait() |
| interrupt() | 中断线程 | 设置中断标志，不强制停止 |
| yield() | 让出 CPU | 只是建议，不保证生效 |
| daemon | 守护线程 | JVM 退出不等待守护线程 |

### 1.3 sleep vs wait

| 特性 | sleep | wait |
|------|-------|------|
| 所属类 | Thread | Object |
| 释放锁 | 不释放 | 释放 |
| 使用位置 | 任意 | synchronized 块内 |
| 唤醒方式 | 超时自动 | notify/notifyAll |
| 用途 | 暂停执行 | 线程间通信 |

## 二、线程池

### 2.1 ThreadPoolExecutor 核心参数

```java
public ThreadPoolExecutor(
    int corePoolSize,        // 核心线程数（即使空闲也不回收）
    int maximumPoolSize,     // 最大线程数
    long keepAliveTime,      // 非核心线程空闲存活时间
    TimeUnit unit,           // 时间单位
    BlockingQueue<Runnable> workQueue,  // 任务队列
    ThreadFactory threadFactory,        // 线程工厂
    RejectedExecutionHandler handler    // 拒绝策略
)
```

### 2.2 执行流程

```
┌──────────────────────────────────────────────────────────────┐
│                    线程池执行流程                              │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│  提交任务                                                      │
│     │                                                         │
│     ▼                                                         │
│  当前线程数 < corePoolSize？                                   │
│     ├── YES ──▶ 创建核心线程执行任务                           │
│     └── NO                                                     │
│          │                                                     │
│          ▼                                                     │
│     工作队列未满？                                              │
│        ├── YES ──▶ 任务放入队列等待                            │
│        └── NO                                                  │
│             │                                                  │
│             ▼                                                  │
│        当前线程数 < maximumPoolSize？                          │
│           ├── YES ──▶ 创建非核心线程执行任务                    │
│           └── NO                                               │
│                │                                               │
│                ▼                                               │
│           执行拒绝策略                                          │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

### 2.3 四种拒绝策略

| 策略 | 行为 | 场景 |
|------|------|------|
| AbortPolicy（默认） | 抛 RejectedExecutionException | 需感知任务丢失 |
| CallerRunsPolicy | 提交线程自己执行 | 不丢弃任务，降速 |
| DiscardPolicy | 静默丢弃 | 允许丢失 |
| DiscardOldestPolicy | 丢弃队列最老任务 | 新任务优先 |

### 2.4 队列选择

| 队列类型 | 特点 | 适用场景 |
|----------|------|----------|
| LinkedBlockingQueue | 无界（默认 Integer.MAX_VALUE） | 一般场景，注意 OOM |
| ArrayBlockingQueue | 有界 | 需要控制队列大小 |
| SynchronousQueue | 不存储，直接交付 | 高吞吐，CachedThreadPool |
| PriorityBlockingQueue | 优先级排序 | 任务有优先级 |
| DelayQueue | 延迟取出 | 定时任务 |

### 2.5 线程数配置

```
CPU 密集型（计算、加密、压缩）：
  corePoolSize = CPU 核数 + 1

IO 密集型（网络、数据库、文件）：
  corePoolSize = CPU 核数 * 2
  或
  corePoolSize = CPU 核数 / (1 - 阻塞系数)
  阻塞系数 ≈ 0.8 ~ 0.9（大多数时间在等 IO）

经验值：
  ┌──────────────┬────────────────────────┐
  │ 场景          │ 核心线程数              │
  ├──────────────┼────────────────────────┤
  │ 纯计算       │ CPU + 1                │
  │ Web 服务     │ CPU * 2 ~ CPU * 4      │
  │ 数据库查询   │ CPU * 4 ~ CPU * 8      │
  │ 混合型       │ 压测调优，先大后小      │
  └──────────────┴────────────────────────┘
```

**面试题：为什么不推荐 Executors 创建线程池？**

| 方法 | 问题 |
|------|------|
| newFixedThreadPool | LinkedBlockingQueue 无界，任务堆积 OOM |
| newSingleThreadExecutor | 同上，无界队列 |
| newCachedThreadPool | maximumPoolSize = Integer.MAX_VALUE，线程爆炸 |
| newScheduledThreadPool | DelayedWorkQueue 无界 |

## 三、锁机制

### 3.1 synchronized

```java
// 三种使用方式
// 1. 修饰实例方法 —— 锁 this
public synchronized void method() { }

// 2. 修饰静态方法 —— 锁 Class 对象
public static synchronized void method() { }

// 3. 修饰代码块 —— 锁指定对象
public void method() {
    synchronized (lockObj) { }
}
```

**synchronized 锁升级**：

```
┌──────────────────────────────────────────────────────────────┐
│                  synchronized 锁升级过程                       │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│  无锁                                                          │
│   │  第一个线程访问                                             │
│   ▼                                                           │
│  偏向锁（Biased Lock）                                         │
│   │  Mark Word 记录线程 ID，无竞争直接重入                      │
│   │  有第二个线程竞争                                           │
│   ▼                                                           │
│  轻量级锁（Lightweight Lock）                                  │
│   │  CAS 自旋竞争，不阻塞                                      │
│   │  自旋失败（竞争激烈）                                       │
│   ▼                                                           │
│  重量级锁（Heavyweight Lock）                                  │
│   │  操作系统互斥量，线程阻塞/唤醒                               │
│   │                                                           │
│   注意：锁只能升级，不能降级（GC 时可批量撤销偏向锁）            │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

### 3.2 ReentrantLock

```java
ReentrantLock lock = new ReentrantLock();
// 公平锁：new ReentrantLock(true)

lock.lock();
try {
    // 临界区
} finally {
    lock.unlock();  // 必须在 finally 中释放
}
```

**synchronized vs ReentrantLock**：

| 特性 | synchronized | ReentrantLock |
|------|-------------|---------------|
| 实现 | JVM 内置 | JDK API |
| 锁获取 | 自动 | 手动 lock/unlock |
| 公平性 | 非公平 | 可选公平/非公平 |
| 可中断 | 不可 | lockInterruptibly() |
| 超时获取 | 不可 | tryLock(timeout) |
| 条件变量 | wait/notify（一个） | Condition（多个） |
| 锁绑定 | 对象 | Condition 对象 |

### 3.3 volatile

```
volatile 两大作用：

1. 保证可见性
   ┌─────────────────────────────────────────┐
   │ 线程A 修改 volatile 变量                  │
   │    → 强制刷回主内存                       │
   │    → 其他线程缓存行失效                   │
   │    → 重新从主内存读取                     │
   └─────────────────────────────────────────┘

2. 禁止指令重排序
   ┌─────────────────────────────────────────┐
   │ DCL 单例中的双重检查：                     │
   │                                          │
   │ 没有 volatile 时：                        │
   │   1. 分配内存                             │
   │   2. 初始化对象  ← 可能被重排到 3 之后     │
   │   3. 指向内存地址                         │
   │                                          │
   │ 其他线程可能在 2 执行前看到非空引用         │
   │ 导致使用未初始化的对象                     │
   └─────────────────────────────────────────┘
```

**volatile 不保证原子性**：

```java
// volatile int count = 0;
// count++ 不是原子操作：
//   1. 读取 count
//   2. count + 1
//   3. 写回 count
// 两个线程同时读到 count=0，各自+1，写回1而不是2

// 解决方案：
// 1. AtomicInteger（CAS）
// 2. synchronized
// 3. LongAdder（分段CAS，高并发更优）
```

## 四、并发工具

### 4.1 AQS

```
┌──────────────────────────────────────────────────────────────┐
│              AQS（AbstractQueuedSynchronizer）                │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│  核心数据结构：                                                │
│  ├── state（volatile int）    同步状态                        │
│  │     ReentrantLock: 0=未锁, 1=已锁, >1=重入                │
│  │     CountDownLatch: 初始值N, countDown到0                  │
│  │     Semaphore: 许可数                                      │
│  │                                                            │
│  └── CLH 双向等待队列                                         │
│     head ◀──▷ Node ◀──▷ Node ◀──▷ Node ◀──▷ tail           │
│              (线程A)    (线程B)    (线程C)                    │
│                                                               │
│  模板方法模式：                                                │
│  ├── tryAcquire()       独占获取（子类实现）                   │
│  ├── tryRelease()       独占释放                              │
│  ├── tryAcquireShared() 共享获取                              │
│  └── tryReleaseShared() 共享释放                              │
│                                                               │
│  基于 AQS 的工具：                                             │
│  ├── ReentrantLock        独占                                │
│  ├── ReentrantReadWriteLock  共享读 + 独占写                  │
│  ├── Semaphore            共享（许可数）                       │
│  ├── CountDownLatch       共享（倒计时）                       │
│  └── CyclicBarrier        不基于AQS（锁条件实现）              │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

### 4.2 并发工具对比

| 工具 | 作用 | 特点 |
|------|------|------|
| CountDownLatch | 等待N个事件完成 | 一次性，不可重置 |
| CyclicBarrier | N个线程互相等待 | 可重置（cyclic） |
| Semaphore | 控制并发数 | 许可证模式 |
| Exchanger | 两个线程交换数据 | 只能在两个线程间 |
| Phaser | 分阶段并行 | 灵活，动态注册 |

### 4.3 ConcurrentHashMap

```
JDK 1.7：Segment 分段锁
  ┌─────────────────────────────────────┐
  │ Segment[] (16段)                    │
  │ ├── Segment 0 → HashEntry[] + lock  │
  │ ├── Segment 1 → HashEntry[] + lock  │
  │ ├── ...                             │
  │ └── Segment 15 → HashEntry[] + lock │
  │                                     │
  │ 并发度 = Segment 数量（默认16）       │
  └─────────────────────────────────────┘

JDK 1.8：CAS + synchronized
  ┌─────────────────────────────────────┐
  │ Node[] (桶数组)                      │
  │ ├── Node 0 → 链表 / TreeBin(红黑树)  │
  │ ├── Node 1 → ...                    │
  │ ├── ...                             │
  │                                     │
  │ 插入逻辑：                          │
  │ 1. 桶为空 → CAS 插入                │
  │ 2. 桶非空 → synchronized(头节点) 插入│
  │ 3. 链表长度 ≥ 8 → 转红黑树           │
  │                                     │
  │ size()：baseCount + CounterCell[]   │
  │ （分散计数，减少竞争）                │
  └─────────────────────────────────────┘
```

## 五、CompletableFuture

```java
// 创建异步任务
CompletableFuture<String> future = CompletableFuture.supplyAsync(() -> {
    return "hello";
}, executor);

// 链式转换
CompletableFuture<String> result = future
    .thenApply(s -> s + " world")          // 转换结果
    .thenCompose(s -> fetchDetail(s))       // 组合另一个 Future
    .thenCombine(otherFuture, (a, b) -> a + b)  // 合并两个 Future
    .exceptionally(e -> "fallback");        // 异常处理

// 多任务编排
CompletableFuture.allOf(f1, f2, f3).join();    // 全部完成
CompletableFuture.anyOf(f1, f2, f3).join();    // 任一完成

// 超时控制（JDK 9+）
future.orTimeout(5, TimeUnit.SECONDS)
      .completeOnTimeout("default", 5, TimeUnit.SECONDS);
```

## 六、ThreadLocal

```java
// 线程局部变量，每个线程独立副本
private static final ThreadLocal<SimpleDateFormat> dateFormat =
    ThreadLocal.withInitial(() -> new SimpleDateFormat("yyyy-MM-dd"));

// 使用
SimpleDateFormat sdf = dateFormat.get();
sdf.format(new Date());

// 用完必须清理，防止内存泄漏
dateFormat.remove();
```

**ThreadLocal 内存泄漏**：

```
┌──────────────────────────────────────────────────────────────┐
│                 ThreadLocal 内存泄漏原理                       │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│  Thread 对象                                                  │
│    └── ThreadLocalMap (弱引用 key, 强引用 value)               │
│          ├── Entry(弱引用:ThreadLocal, 强引用:value)           │
│          └── ...                                              │
│                                                               │
│  Key 被 GC 回收后，value 仍被强引用无法回收                     │
│  → 线程池场景下线程复用，value 累积导致泄漏                    │
│                                                               │
│  解决：                                                       │
│  1. 用完一定要 remove()                                       │
│  2. 线程池中尤其注意                                          │
│                                                               │
│  InheritableThreadLocal：子线程可继承父线程的值                 │
│  线程池场景用 TransmittableThreadLocal（阿里开源）              │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

## 七、高频面试题

| 问题 | 核心答案 |
|------|----------|
| 线程池核心参数？执行顺序？ | corePool → queue → maxPool → reject |
| synchronized 锁升级过程？ | 无锁 → 偏向锁 → 轻量级锁 → 重量级锁 |
| volatile 的作用和局限？ | 保证可见性+禁止重排序，不保证原子性 |
| AQS 原理？ | state + CLH队列 + CAS + 模板方法 |
| ConcurrentHashMap 1.7 vs 1.8？ | 1.7 Segment分段锁；1.8 CAS+synchronized+红黑树 |
| ThreadLocal 内存泄漏？ | Key弱引用被GC后value强引用残留，必须remove() |
| CountDownLatch vs CyclicBarrier？ | 前者一次性倒计数，后者可重置循环屏障 |
| CompletableFuture 怎么做超时？ | orTimeout() / completeOnTimeout()（JDK9+） |
