## 一、字节码基础

### 1.1 Class 文件结构

```
┌──────────────────────────────────────────────────────────┐
│                  Class 文件格式                           │
├──────────────────────────────────────────────────────────┤
│                                                           │
│  ClassFile {                                              │
│    u4             magic;           // 0xCAFEBABE         │
│    u2             minor_version;                         │
│    u2             major_version;    // JDK8=52, JDK17=61 │
│    u2             constant_pool_count;                    │
│    cp_info        constant_pool[constant_pool_count-1];   │
│    u2             access_flags;     // public/final等     │
│    u2             this_class;                             │
│    u2             super_class;                            │
│    u2             interfaces_count;                       │
│    u2             interfaces[interfaces_count];           │
│    u2             fields_count;                           │
│    field_info     fields[fields_count];                   │
│    u2             methods_count;                          │
│    method_info    methods[methods_count];                 │
│    u2             attributes_count;                       │
│    attribute_info attributes[attributes_count];           │
│  }                                                        │
│                                                           │
└──────────────────────────────────────────────────────────┘
```

### 1.2 字节码指令分类

| 类别 | 典型指令 | 说明 |
|------|----------|------|
| 加载存储 | aload_0, iload_1, astore_2 | 局部变量 ↔ 操作数栈 |
| 算术 | iadd, isub, imul | 整数运算 |
| 类型转换 | i2l, l2d | 类型转换 |
| 对象操作 | new, invokespecial, getfield | 对象创建和访问 |
| 方法调用 | invokevirtual, invokeinterface, invokestatic, invokespecial | 四种调用 |
| 控制流 | ifeq, goto, tableswitch | 条件跳转 |
| 返回 | ireturn, areturn, return | 方法返回 |

### 1.3 方法调用指令

```
┌──────────────────────────────────────────────────────────┐
│                  四种方法调用指令                           │
├──────────────────────────────────────────────────────────┤
│                                                           │
│  invokestatic      调用静态方法                            │
│  invokespecial     调用构造方法/private/super 方法         │
│  invokevirtual     调用虚方法（运行时多态）                │
│  invokeinterface   调用接口方法                            │
│  invokedynamic     动态调用（Lambda/Scala/Groovy）         │
│                                                           │
│  编译时可知：invokestatic / invokespecial                  │
│  运行时确定：invokevirtual / invokeinterface               │
│                                                           │
└──────────────────────────────────────────────────────────┘
```

```
┌──────────────────────────────────────────────────────────┐
│                Java Agent 两种加载方式                     │
├──────────────────────────────────────────────────────────┤
│                                                           │
│  启动时加载（premain）：                                   │
│  java -javaagent:myagent.jar -jar app.jar                │
│                                                           │
│  public static void premain(String args,                 │

│              String className, Class<?> classBeingRedf,  │
│              ProtectionDomain domain, byte[] classfile) { │
│              // 修改字节码                                │
│              return modifiedBytes;                        │
│          }                                                │
│      });                                                  │
│  }                                                        │
│                                                           │
│  运行时加载（agentmain）：                                 │
│  VirtualMachine vm = VirtualMachine.attach(pid);          │
│  vm.loadAgent("myagent.jar");                             │
│                                                           │
│  public static void agentmain(String args,               │
│      Instrumentation inst) {                              │
│      inst.retransformClasses(targetClass);                │
│  }                                                        │
│                                                           │

```

## 三、字节码操作框架

### 3.1 框架对比

| 框架 | API 风格 | 性能 | 学习曲线 | 典型用户 |
|------|----------|------|----------|----------|
| ASM | 低层访问者模式 | 最高 | 陡峭 | Spring / CGLIB / JDK Lambda |
| Javassist | 源码级 API | 中 | 平缓 | Dubbo / MyBatis |
| Byte Buddy | 流式 API | 高 | 简单 | Mockito / SkyWalking |
| CGLIB | 高层封装 | 中 | 中 | Spring AOP（旧版） |

### 3.2 ASM 核心

```
┌──────────────────────────────────────────────────────────┐
│                    ASM 核心模型                            │
├──────────────────────────────────────────────────────────┤
│                                                           │
│  ClassReader       读取 .class 文件字节码                  │
│       │                                                   │
│       ▼                                                   │
│  ClassVisitor      访问类结构（字段/方法/注解）             │
│       │                                                   │
│       ├── FieldVisitor     访问字段                       │
│       ├── MethodVisitor    访问方法（核心）                │
│       └── AnnotationVisitor 访问注解                      │
│                                                           │
│       │                                                   │
│       ▼                                                   │
│  ClassWriter       生成修改后的字节码                      │
│                                                           │
│  示例：方法耗时统计                                        │
│  ┌────────────────────────────────────────────┐           │
│  │ 原始方法：                                  │           │
│  │   public void doSomething() {               │           │
│  │       // 业务逻辑                           │           │
│  │   }                                         │           │
│  │                                             │           │
│  │ 增强后：                                     │           │
│  │   public void doSomething() {               │           │
│  │       long start = System.currentTimeMillis();│          │
│  │       try {                                 │           │
│  │           // 业务逻辑                       │           │
│  │       } finally {                           │           │
│  │           long cost = System.currentTimeMillis()│        │
│  │               - start;                      │           │
│  │           System.out.println("cost:" + cost);│          │
│  │       }                                     │           │
│  │   }                                         │           │
│  └────────────────────────────────────────────┘           │
│                                                           │
└──────────────────────────────────────────────────────────┘
```

**ASM MethodVisitor 关键方法**：

```java
// ASM 方法增强示例（伪代码）
public class TimingMethodVisitor extends MethodVisitor {

    @Override
    public void onMethodEnter() {
        // 方法入口：插入 long start = System.currentTimeMillis();
        mv.visitMethodInsn(INVOKESTATIC, "java/lang/System", "currentTimeMillis", "()J", false);
        mv.visitVarInsn(LSTORE, startTimeVar);
    }

    @Override
    public void onMethodExit(int opcode) {
        // 方法出口：插入耗时打印（finally 逻辑）
        if (opcode != ATHROW) {
            mv.visitMethodInsn(INVOKESTATIC, "java/lang/System", "currentTimeMillis", "()J", false);
            mv.visitVarInsn(LLOAD, startTimeVar);
            mv.visitInsn(LSUB);
            // ... 打印逻辑
        }
    }
}
```

### 3.3 Byte Buddy 示例

```java
// Byte Buddy 流式 API
new AgentBuilder.Default()
    .type(ElementMatchers.named("com.example.OrderService"))
    .transform((builder, typeDescription, classLoader, module) ->
        builder.method(ElementMatchers.named("createOrder"))
               .intercept(Advice.to(TimingAdvice.class))
    ).installOn(inst);

// Advice 注解方式
public class TimingAdvice {

    @Advice.OnMethodEnter
    public static long onEnter() {
        return System.currentTimeMillis();
    }

    @Advice.OnMethodExit(onThrowable = Throwable.class)
    public static void onExit(
        @Advice.Enter long startTime,
        @Advice.Thrown Throwable throwable
    ) {
        long cost = System.currentTimeMillis() - startTime;
        System.out.println("Method cost: " + cost + "ms");
    }
}
```

### 3.4 Javassist 示例

```java
// Javassist 源码级 API（最直观）
ClassPool pool = ClassPool.getDefault();
CtClass cc = pool.get("com.example.OrderService");
CtMethod method = cc.getDeclaredMethod("createOrder");

// 在方法开头插入代码
method.insertBefore("{ long start = System.currentTimeMillis(); }");

// 在方法末尾插入代码
method.insertAfter("{ System.out.println(\"cost:\" + (System.currentTimeMillis() - start)); }");

// 生成增强后的类
cc.toClass();
```

## 四、动态代理与字节码增强

### 4.1 CGLIB 原理

```
┌──────────────────────────────────────────────────────────┐
│                  CGLIB 实现原理                           │
├──────────────────────────────────────────────────────────┤
│                                                           │
│  1. 生成目标类的子类（ASM 生成字节码）                     │
│  2. 子类重写所有非 final 方法                              │
│  3. 重写的方法调用 MethodInterceptor.intercept()          │
│  4. intercept() 中通过 MethodProxy 调用原始方法            │
│                                                           │
│  FastClass 机制：                                          │
│  ├── 为目标类和代理类各生成一个 FastClass                  │
│  ├── 方法调用通过索引定位，不走反射                        │
│  └── 比反射调用 JDK 动态代理更快                           │
│                                                           │
│  限制：                                                    │
│  ├── 无法代理 final 类和 final 方法                       │
│  ├── 无法代理 private 方法                                │
│  └── 无法代理 static 方法                                 │
│                                                           │
└──────────────────────────────────────────────────────────┘
```

### 4.2 Spring AOP 代理创建时机

```
┌──────────────────────────────────────────────────────────┐
│           Spring AOP 代理创建流程（AbstractAutoProxyCreator）│
├──────────────────────────────────────────────────────────┤
│                                                           │
│  Bean 初始化后                                             │
│  → BeanPostProcessor.postProcessAfterInitialization()    │
│  → wrapIfNecessary()                                     │
│  → getAdvicesAndAdvisorsForBean()  获取切面               │
│  → createProxy()                                         │
│     ├── 有接口？                                          │
│     │   ├── 默认 JDK 动态代理                              │
│     │   └── spring.aop.proxy-target-class=true → CGLIB   │
│     └── 无接口 → CGLIB                                   │
│  → 返回代理对象                                           │
│                                                           │
│  Spring Boot 2.x 默认 proxy-target-class=true (CGLIB)    │
│                                                           │
└──────────────────────────────────────────────────────────┘
```

## 五、热部署与热加载

### 5.1 热加载 vs 热部署

| 特性 | 热加载 | 热部署 |
|------|--------|--------|
| 机制 | 修改方法体，替换 Class | 重新创建 ClassLoader |
| 范围 | 单个类 | 整个应用上下文 |
| 速度 | 快 | 慢 |
| 状态 | 保留运行状态 | 重置状态 |
| 代表 | JRebel | Spring DevTools |

### 5.2 类热替换的局限

```
┌──────────────────────────────────────────────────────────┐
│              类热替换的限制                                │
├──────────────────────────────────────────────────────────┤
│                                                           │
│  Instrumentation.redefineClasses() 限制：                 │
│  ├── 不能添加/删除/重命名字段                              │
│  ├── 不能添加/删除/重命名方法                              │
│  ├── 不能改变类的继承关系                                  │
│  ├── 不能改变方法签名                                     │
│  └── 只能修改方法体                                       │
│                                                           │
│  Instrumentation.retransformClasses() 限制：              │
│  └── 同上，但可以恢复原始字节码                            │
│                                                           │
│  完整热替换方案：                                          │
│  ├── 新建 ClassLoader 加载新版本类                        │
│  ├── 将引用逐步切换到新类                                 │
│  └── 旧 ClassLoader 和旧类等待 GC                        │
│      → OSGi / Tomcat 的实现方式                           │
│                                                           │
└──────────────────────────────────────────────────────────┘
```

## 六、高频面试题

| 问题 | 核心答案 |
|------|----------|
| Java Agent 的原理？ | 基于 Instrumentation API，通过 premain/agentmain 在类加载时/运行时修改字节码 |
| ASM 和 Javassist 怎么选？ | ASM 性能最好适合底层框架；Javassist API 简单适合业务级增强 |
| CGLIB 为什么不能代理 final？ | CGLIB 通过生成子类重写方法实现，final 方法不可重写 |
| Spring AOP 什么时候用 JDK 代理什么时候用 CGLIB？ | 有接口时默认 JDK，Spring Boot 2.x 默认全用 CGLIB |
| SkyWalking 是怎么实现无侵入监控的？ | Java Agent + Byte Buddy 字节码增强，拦截方法调用注入 Trace 逻辑 |
| 热部署的原理？ | 重新创建 ClassLoader 加载新类，或 Instrumentation retransform |
| invokedynamic 是什么？ | JDK7 新增指令，支持运行时动态确定调用目标，Lambda 底层使用 |
