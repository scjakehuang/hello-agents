## 一、六大原则

| 原则 | 含义 | 举例 |
|------|------|------|
| 单一职责（SRP） | 一个类只做一件事 | UserService 不处理订单逻辑 |
| 开闭原则（OCP） | 对扩展开放，对修改关闭 | 策略模式加新策略不改旧代码 |
| 里氏替换（LSP） | 子类可替换父类 | 子类不重写父类非抽象方法 |
| 接口隔离（ISP） | 接口最小化 | 不设计大而全的接口 |
| 依赖倒置（DIP） | 依赖抽象不依赖实现 | @Autowired 注入接口而非实现 |
| 迪米特法则（LOD） | 最少知道 | 对象间最少交互 |

## 二、高频面试设计模式

### 2.1 单例模式

```java
// 方式一：双重检查锁（DCL）
public class Singleton {
    private static volatile Singleton instance;  // volatile 防止指令重排

    private Singleton() {}

    public static Singleton getInstance() {
        if (instance == null) {                 // 第一次检查（避免不必要的加锁）
            synchronized (Singleton.class) {
                if (instance == null) {         // 第二次检查（防止重复创建）
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}

// 方式二：静态内部类（推荐，延迟加载+线程安全）
public class Singleton {
    private Singleton() {}

    private static class Holder {
        private static final Singleton INSTANCE = new Singleton();
    }

    public static Singleton getInstance() {
        return Holder.INSTANCE;
    }
}

// 方式三：枚举（最安全，防反射/序列化破坏）
public enum Singleton {
    INSTANCE;

    public void doSomething() { }
}
```

**面试题：单例模式如何防止反射破坏？**

```java
// 构造器中检查
private Singleton() {
    if (Holder.INSTANCE != null) {
        throw new RuntimeException("不允许反射创建实例");
    }
}
// 枚举天然防反射（JVM 保证枚举单例）
```

### 2.2 策略模式

```java
// 消除 if-else / switch 的利器

// 策略接口
public interface PayStrategy {
    PayResult pay(PayRequest request);
}

// 策略实现
@Component("alipay")
public class AlipayStrategy implements PayStrategy {
    @Override
    public PayResult pay(PayRequest request) { /* 支付宝支付 */ }
}

@Component("wechat")
public class WechatPayStrategy implements PayStrategy {
    @Override
    public PayResult pay(PayRequest request) { /* 微信支付 */ }
}

// 策略工厂（Spring 自动注入）
@Service
public class PayStrategyFactory {
    @Autowired
    private Map<String, PayStrategy> strategyMap;  // Key=Bean名

    public PayStrategy getStrategy(String payType) {
        PayStrategy strategy = strategyMap.get(payType);
        if (strategy == null) {
            throw new IllegalArgumentException("不支持的支付方式: " + payType);
        }
        return strategy;
    }
}

// 使用
payStrategyFactory.getStrategy("alipay").pay(request);
```

```java
// Spring 事件机制就是观察者模式

// 1. 定义事件
public class OrderCreatedEvent extends ApplicationEvent {
    private final Order order;
    public OrderCreatedEvent(Object source, Order order) {
        super(source);

    }

// 2. 发布事件
@Service
public class OrderService {
    @Autowired

    public void createOrder(Order order) {
        // 创建订单...

// 3. 监听事件
```

### 2.5 模板方法模式

```java
// 定义算法骨架，子类实现具体步骤

public abstract class AbstractDataImporter {

    // 模板方法（final 防止重写）
    public final void importData(File file) {
        String content = readFile(file);
        List<Data> data = parse(content);
        validate(data);
        save(data);
        notifyComplete();
    }

    protected abstract List<Data> parse(String content);
    protected abstract void save(List<Data> data);

    // 公共步骤
    private String readFile(File file) { /* 读取文件 */ }
    private void validate(List<Data> data) { /* 校验数据 */ }
    private void notifyComplete() { /* 通知完成 */ }
}

// Spring 中的模板方法：
// AbstractApplicationContext.refresh() → 模板方法
// JdbcTemplate / RedisTemplate → 模板模式
```

### 2.6 责任链模式

```java
// 请求沿链传递，每个节点决定是否处理/传递

public abstract class Handler {
    protected Handler next;

    public Handler setNext(Handler next) {
        this.next = next;
        return next;
    }

    public abstract void handle(Request request);
}

// Spring 中的责任链：
// Filter Chain / Interceptor Chain
// Spring Security FilterChain

// 业务示例：风控规则链
@Component
public class RiskCheckChain {
    private final List<RiskHandler> handlers;

    public RiskCheckChain(List<RiskHandler> handlers) {
        this.handlers = handlers;
    }

    public RiskResult check(Order order) {
        for (RiskHandler handler : handlers) {
            RiskResult result = handler.check(order);
            if (result.isBlocked()) {
                return result;  // 命中规则，中断链
            }
        }
        return RiskResult.pass();
    }
}
```

### 2.7 代理模式

```java
// 静态代理
public class UserDaoProxy implements UserDao {
    private UserDao target;

    @Override
    public void save() {
        System.out.println("开始事务");
        target.save();
        System.out.println("提交事务");
    }
}

// JDK 动态代理（接口）
Object proxy = Proxy.newProxyInstance(
    target.getClass().getClassLoader(),
    target.getClass().getInterfaces(),
    (proxy, method, args) -> {
        before();
        Object result = method.invoke(target, args);
        after();
        return result;
    }
);

// CGLIB 动态代理（子类继承）
Enhancer enhancer = new Enhancer();
enhancer.setSuperclass(targetClass);
enhancer.setCallback((obj, method, args, proxy) -> {
    before();
    Object result = proxy.invokeSuper(obj, args);
    after();
    return result;
});

// Spring AOP 代理 = 代理模式 + 动态代理技术
```

## 四、高频面试题

| 问题 | 核心答案 |
|------|----------|
| 单例模式有几种实现？ | 饿汉、DCL（volatile）、静态内部类、枚举（最安全） |
| 策略模式解决什么问题？ | 消除 if-else / switch，新策略加类不改旧代码 |
| JDK 动态代理和 CGLIB 的区别？ | JDK 基于接口，CGLIB 基于继承；Spring Boot 默认全 CGLIB |
| Spring 用了哪些设计模式？ | 工厂（BeanFactory）、单例（Bean）、代理（AOP）、模板（JdbcTemplate）、观察者（Event）等 |
| 责任链和装饰者模式的区别？ | 责任链侧重传递和中断，装饰者侧重增强且不中断 |
| 开闭原则在实际项目中怎么体现？ | 策略/工厂/观察者模式，扩展新功能只加类不改旧代码 |
