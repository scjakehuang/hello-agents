## 四、第三阶段：Agent 关键技术

### 1. RAG 检索增强生成

把LLM基础和核心组件吃透之后，接下来就要攻克Agent工程落地最核心、面试必考的关键技术——RAG检索增强生成，这部分也是Java后端同学转Agent最容易建立优势、快速做出落地成果的模块，哪怕是刚接触AI领域，靠着后端已有的存储、检索、接口开发功底，也能快速上手，不用死磕复杂算法和模型原理，完全聚焦工程实战。

#### 为什么需要 RAG？

很多后端同学刚接触Agent会疑惑，已经学会了大模型调用和工具调用，为什么非要单独学RAG，其实核心原因在于原生大模型存在两个无法规避的致命缺陷，直接导致纯LLM搭建的Agent只能停留在Demo阶段，根本无法满足企业级场景的落地要求：

| 问题 | 说明 |
|------|------|
| **训练数据固定滞后** | 对于企业内部的私有文档、实时业务数据、最新行业资讯、内部知识库这类未进入训练集的内容，模型完全没有认知 |
| **幻觉问题** | 会在没有依据的情况下编造信息、扭曲事实，放到企业场景里，轻则导致回答错误影响业务，重则引发合规风险和客户投诉 |

RAG就是解决这两大问题的最优解：它不需要微调模型、不改动模型权重，只通过外部知识库检索真实可信的信息，再把检索到的事实依据和用户问题一起交给大模型，让模型基于现有资料生成回答，从根源上杜绝幻觉，同时实现实时知识更新，兼顾安全性和实用性。

#### RAG 核心工作流程

对于后端开发者来说，RAG本质上就是把我们熟悉的数据库查询、搜索引擎检索、缓存优化等技术，和大模型生成能力结合起来，完全契合后端的技术思维。

| 步骤 | 说明 | 后端类比 |
|------|------|----------|
| 1. 文档加载与预处理 | 把PDF、Word、Excel、markdown文档统一读取，完成格式清洗和文本规整 | 数据导入、文件处理 |
| 2. 文本切片处理 | 把过长文本拆分成合适大小的文本块，设置重叠窗口避免语义断裂 | 数据分片、分页处理 |
| 3. 向量化嵌入 | 通过嵌入模型把文本块转换成数值向量 | 调用模型接口 |
| 4. 检索召回 | 用户提问转换成向量，通过相似度计算匹配最相关文本片段 | 索引查询、精准检索 |
| 5. 提示词拼接与生成 | 检索到的上下文和用户问题整合进提示词，调用大模型生成回答 | 业务逻辑处理 |

### 企业级 RAG vs Demo 区别

这里推荐的项目教程，核心作用是帮大家快速入门、理清基础流程，本质上都是单机轻量化Demo，只覆盖了核心链路的基础演示，完全达不到企业生产环境的上线标准。

对于后端转Agent的开发者来说，企业级RAG系统从来不是简单的文档切片加向量检索再加模型生成，而是要结合成熟的后端工程经验，补齐高可用、高并发、数据安全、性能优化、稳定性兜底等全套生产级能力。

#### 企业级 RAG 关键点

| 维度 | Demo | 企业级 |
|------|------|--------|
| **数据处理** | 几份干净文档 | 海量杂乱非结构化数据（PDF扫描件、复杂表格、加密文件等） |
| **性能优化** | 单机测试 | Redis缓存、向量库索引优化、线程池、异步调用、限流熔断 |
| **准确率** | 基础检索 | 混合检索、语义重排、多路召回策略、兜底机制 |
| **监控** | 无 | 全链路日志监控、追踪检索结果、响应耗时、异常信息 |
| **扩展性** | 固定方案 | 支持模型、向量库、嵌入模型灵活切换 |
